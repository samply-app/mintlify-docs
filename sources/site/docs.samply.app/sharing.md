# Sharing [​](https://docs.samply.app/#sharing)

We believe creators should have full control over how their audio is shared. We prioritize security, ensuring your work-in-progress tracks remain protected at all times.

## Private Projects [​](https://docs.samply.app/#private-projects)

The most secure way to share tracks is by inviting people to your private project. Before accessing your files, they must create a free Samply account and verify their email, ensuring authenticity.

![Share Dialog](https://docs.samply.app/img/sharing/share-dialog.png)

Setting your project to **Private** guarantees that only verified and invited users can access your files. Invited people can have one of three roles:

| Role | Permissions |
| --- | --- |
| Admin | Delete projects. Edit artwork. Edit name. Invite others. |
| Editor | Upload files. Delete files. Rename files. Edit link options. |
| Listener | Listen to all files in a project. Additional access (comments, downloads, versions) subject to [Access Options](https://docs.samply.app/#access-options) |

### Invite via Email [​](https://docs.samply.app/#invite-via-email)

Tap the **Share** button or press `.` to open the **Share Dialog**. Enter your invitee's email address, select their access level, and press **Enter**. Once they accept the invite, they’ll be saved in your address book for quicker access in the future.

### Invite Links [​](https://docs.samply.app/#invite-links)

If you don’t have their email, you can generate an invite link. This link will expire after it’s used, keeping your project secure.

### Access Requests [​](https://docs.samply.app/#access-requests)

If someone receives a private project link without access, they’ll see an **Access Denied** screen with an option to request access. You’ll be notified and can choose to grant or deny the request.

## Public Projects [​](https://docs.samply.app/#public-projects)

If convenience is more important than security, you can set your project to **Public**. When you share a link to a public project, listeners can start listening immediately—no account required.

### Access Options [​](https://docs.samply.app/#access-options)

You can control your listeners' experience further by modifying access options such as adding a password or disabling downloads. These can be configured via the [Options Panel](https://docs.samply.app/./overview.html#options-panel).

![Access Options](https://docs.samply.app/img/sharing/access-options.png)

#### Comments [​](https://docs.samply.app/#comments)

Disable commenting on shared links if needed.

#### Downloads [​](https://docs.samply.app/#downloads)

Prevent listeners from downloading your tracks while still allowing playback.

#### Versions [​](https://docs.samply.app/#versions)

Restrict version switching so listeners can only access the latest version of your tracks.

#### Passwords [​](https://docs.samply.app/#passwords)

Add a password for extra security. For best practice, share the password separately from the link.

#### Payments [​](https://docs.samply.app/#payments)

Set up a paywall to restrict access to or downloads of your files.

### Additional Links [​](https://docs.samply.app/#additional-links)

Beyond the primary project link, you can generate links for specific files, folders, or multiple items. Each link is unique and can have it's access options customized independently. Manage your links in the [Options Panel](https://docs.samply.app/./overview.html#options-panel) under the **Links** section, where you can delete or deactivate them at any time.
