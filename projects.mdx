Every file in Samply lives inside a project. Each project has its own file structure and user access settings, making it easy to keep work organized and share it on your own terms. Understanding how projects work helps you get the most out of your Samply workflow.

## View Modes

Projects offer two view modes designed for different tasks. Switch between them using the toggle in the **Toolbar** or by pressing `\` on your keyboard.

<Tabs>
  <Tab title="Listen Mode">
    Listen Mode presents a clean, streamlined interface focused entirely on playback. This is exactly what your listeners see when you share a public link — no distracting metadata, just the music.

    Use Listen Mode to preview your project exactly as an outside listener would experience it before sending it out.
  </Tab>
  <Tab title="Manage Mode">
    Manage Mode is a detailed, data-rich view built for organizing your files. It surfaces metadata columns, file details, and management controls so you can sort, inspect, and arrange your content efficiently.

    Use Manage Mode when you're actively working on a project and need full visibility into your files.
  </Tab>
</Tabs>

## Data Columns

In Manage Mode, you control which data columns are visible so you can focus on what matters most. To customize column visibility, tap the **3-dot menu** next to the column headers or right-click anywhere on the column header row and adjust the settings from the menu that appears.

## Billing & Ownership

<Info>
  The **project owner** is always the user who originally created the project. Storage usage for all files in a project counts toward the owner's storage quota, not the quota of collaborators.
</Info>

If you need to hand a project off to someone else, you can transfer ownership through the **Share Dialog**:

<Steps>
  <Step title="Open the Share Dialog">
    Click the share button for the project you want to transfer.
  </Step>
  <Step title="Select Transfer Ownership">
    Choose **Transfer ownership** and select the collaborator you want to make the new owner.
  </Step>
  <Step title="Confirm the transfer">
    The selected collaborator receives an invitation. Once they accept, they gain full ownership and the project storage counts toward their quota.
  </Step>
</Steps>

## Deleting a Project

To delete a project, right-click it in the **Library** and select **Delete**.

<Tip>
  If you'd rather not delete projects manually, enable **Auto-Archive** in the project settings. Auto-Archive automatically moves inactive projects to your archive after a period of time you define, keeping your Library clean without permanently removing anything.
</Tip>

<Warning>
  Deleting a project is permanent. If you're unsure, use Auto-Archive instead to preserve access to your files.
</Warning>
