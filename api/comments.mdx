The Comments API lets you programmatically create and retrieve feedback on individual audio files. You can attach comments to specific moments in a track using audio timestamps, thread replies, flag items as completed, and pin important notes — making it straightforward to integrate Samply's feedback workflow into your own tools and automations.

<Note>
  The `fileid` in all comments endpoints must be the id of a **specific file version** — not a stack id. If you provide a stack id, no comment will be created.
</Note>

## Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/projects/:projectid/files/:fileid/comments` | Create a comment |
| `GET` | `/projects/:projectid/files/:fileid/comments` | List all comments on a file |

---

## Create a comment

`POST /projects/:projectid/files/:fileid/comments`

Creates a new comment on the specified file and returns a [comment object](#comment-object).

### Body parameters

<ParamField body="message" type="string" required>
  The markdown text content of the comment. Best practice is to wrap your message in paragraph tags, e.g. `<p>Sounds great here.</p>`.
</ParamField>

<ParamField body="audioTimestamp" type="number">
  The comment's start time in seconds. Use this to attach the comment to a specific moment in the track. For a single point (not a range), only this field needs to be set.
</ParamField>

<ParamField body="audioTimestampEnd" type="number">
  The comment's end time in seconds. When used together with `audioTimestamp`, this creates a time range annotation. Note that replies do not display timecodes.
</ParamField>

<ParamField body="parentid" type="string">
  The id of the parent comment if this comment is a reply. The parent comment must not itself be a reply (replies cannot be nested).
</ParamField>

<ParamField body="completed" type="boolean">
  Whether to mark this comment as completed immediately upon creation.
</ParamField>

<ParamField body="pinned" type="boolean">
  Whether to pin this comment so it appears prominently in the feedback view.
</ParamField>

### Example

```javascript
const response = await fetch(
  `https://samply.app/api/v0/projects/${projectid}/files/${fileid}/comments`,
  {
    method: "POST",
    headers: {
      Authorization: "Bearer YOUR_ACCESS_TOKEN",
      "Content-Type": "application/json"
    },
    body: JSON.stringify({
      message: "<p>The kick feels a bit too loud here.</p>",
      audioTimestamp: 32.5,
      audioTimestampEnd: 35.0
    })
  }
);
const comment = await response.json();
```

## List comments

`GET /projects/:projectid/files/:fileid/comments`

Returns an array of all [comment objects](#comment-object) for the specified file.

---

## Comment object

All comment endpoints return a comment object with the following structure.

<ResponseField name="id" type="string">
  The unique comment id.
</ResponseField>

<ResponseField name="object" type="string">
  Always `"comment"`.
</ResponseField>

<ResponseField name="message" type="string">
  The markdown text content of the comment.
</ResponseField>

<ResponseField name="audioTimestamp" type="number">
  The comment's start time in seconds. If the comment is not time-coded, this value will be absent.
</ResponseField>

<ResponseField name="audioTimestampEnd" type="number">
  The comment's end time in seconds. Present when the comment spans a time range.
</ResponseField>

<ResponseField name="completed" type="boolean">
  Whether this comment is marked as complete.
</ResponseField>

<ResponseField name="creator" type="object">
  The user who created the comment.

  <Expandable title="creator fields">
    <ResponseField name="displayName" type="string">
      The creator's human-readable display name.
    </ResponseField>
    <ResponseField name="email" type="string">
      The creator's email address.
    </ResponseField>
    <ResponseField name="photoURL" type="string">
      A URL for the creator's profile picture.
    </ResponseField>
    <ResponseField name="uid" type="string">
      The creator's unique user id.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="isReply" type="boolean">
  Whether this comment is a reply to another comment.
</ResponseField>

<ResponseField name="parentid" type="string">
  The id of the parent comment, if this comment is a reply.
</ResponseField>

<ResponseField name="reactions" type="object">
  A map of emoji reactions added to this comment. Each key is an emoji string and each value contains reaction details.

  <Expandable title="reaction fields">
    <ResponseField name="count" type="number">
      The number of times this emoji reaction has been added.
    </ResponseField>
    <ResponseField name="users" type="object">
      A map of users who added this reaction, keyed by user id.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="timeCreated" type="number">
  The time the comment was created, in milliseconds since the Unix epoch.
</ResponseField>

<ResponseField name="timeModified" type="number">
  The time the comment was last modified, in milliseconds since the Unix epoch.
</ResponseField>
