Projects are the top-level containers in Samply. Every file you upload lives inside a project, and every player you create draws from a project's files. Through the Projects API you can create new projects, update their settings, invite collaborators, and retrieve project details programmatically.

## Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/projects` | Create a project |
| `GET` | `/projects` | List all projects you own |
| `GET` | `/projects/:projectid` | Get a project by id |
| `POST` | `/projects/:projectid` | Update a project |

---

## Create a project

`POST /projects`

Creates a new project and returns a [project object](#project-object). As a side effect, Samply also creates a project-scoped player using your default player options — that player automatically includes all files added to the project.

### Body parameters

<ParamField body="name" type="string" required>
  A human-readable title for the project.
</ParamField>

<ParamField body="upload" type="object">
  Options for the project's public upload portal.

  <Expandable title="upload fields">
    <ParamField body="enabled" type="boolean">
      Enable the public upload portal for this project.
    </ParamField>
    <ParamField body="header" type="string">
      The title displayed at the top of the upload portal.
    </ParamField>
    <ParamField body="greeting" type="string">
      A subheader or welcome message displayed on the upload portal.
    </ParamField>
    <ParamField body="redirect" type="string">
      A URL to redirect the user to after their upload completes.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="collaborators" type="array">
  An array of collaborators to invite when the project is created. Each entry requires an email address and a role.

  <Expandable title="collaborator fields">
    <ParamField body="email" type="string" required>
      The collaborator's email address.
    </ParamField>
    <ParamField body="role" type="'admin' | 'editor' | 'viewer'" required>
      The permission level to grant this collaborator on the project.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="sortBy" type="object">
  Default sort order for files within the project.

  <Expandable title="sortBy fields">
    <ParamField body="criterion" type="'custom' | 'name' | 'duration' | 'size' | 'bitDepth' | 'sampleRate' | 'bitRate' | 'channels' | 'lufs' | 'timeModified' | 'timeCreated' | 'kind'">
      The attribute to sort files by.
    </ParamField>
    <ParamField body="ascending" type="boolean">
      Set to `true` to sort in ascending order, `false` for descending.
    </ParamField>
  </Expandable>
</ParamField>

### Example

```javascript
const response = await fetch("https://samply.app/api/v0/projects", {
  method: "POST",
  headers: {
    Authorization: "Bearer YOUR_ACCESS_TOKEN",
    "Content-Type": "application/json"
  },
  body: JSON.stringify({
    name: "Album Mixes - Q4",
    collaborators: [
      { email: "engineer@studio.com", role: "editor" }
    ]
  })
});
const project = await response.json();
```

## Update a project

`POST /projects/:projectid`

Updates an existing project and returns the updated [project object](#project-object). You can modify the project name and upload portal settings. To manage collaborators after creation, use the in-app settings.

### Body parameters

<ParamField body="name" type="string">
  A new human-readable title for the project.
</ParamField>

<ParamField body="upload" type="object">
  Updated options for the project's public upload portal.

  <Expandable title="upload fields">
    <ParamField body="enabled" type="boolean">
      Enable or disable the public upload portal.
    </ParamField>
    <ParamField body="header" type="string">
      The title displayed at the top of the upload portal.
    </ParamField>
    <ParamField body="greeting" type="string">
      A subheader or welcome message displayed on the upload portal.
    </ParamField>
    <ParamField body="redirect" type="string">
      A URL to redirect the user to after their upload completes.
    </ParamField>
  </Expandable>
</ParamField>

## Get a project

`GET /projects/:projectid`

Returns a single [project object](#project-object) for the given project id.

---

## List owned projects

`GET /projects`

Returns an array of all [project objects](#project-object) that you own.

## Project object

All project endpoints return a project object with the following structure.

<ResponseField name="id" type="string">
  The unique project id.
</ResponseField>

<ResponseField name="name" type="string">
  The human-readable title of the project.
</ResponseField>

<ResponseField name="object" type="string">
  Always `"project"`.
</ResponseField>

<ResponseField name="artwork" type="string">
  A download URL for the project's cover artwork.
</ResponseField>

<ResponseField name="color" type="string">
  The project's display color in hex format (e.g. `#a259ff`).
</ResponseField>

<ResponseField name="creator" type="object">
  The user who created the project.

  <Expandable title="creator fields">
    <ResponseField name="displayName" type="string">
      The creator's human-readable display name.
    </ResponseField>
    <ResponseField name="email" type="string">
      The creator's email address.
    </ResponseField>
    <ResponseField name="photoURL" type="string">
      A download URL for the creator's profile picture.
    </ResponseField>
    <ResponseField name="uid" type="string">
      The creator's unique user id.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="size" type="number">
  The total size in bytes of all files contained within the project.
</ResponseField>

<ResponseField name="timeCreated" type="number">
  The time the project was created, in milliseconds since the Unix epoch.
</ResponseField>

<ResponseField name="timeModified" type="number">
  The time the project was last modified, in milliseconds since the Unix epoch.
</ResponseField>

<ResponseField name="upload" type="object">
  The project's upload portal configuration.

  <Expandable title="upload fields">
    <ResponseField name="enabled" type="boolean">
      Whether the public upload portal is enabled.
    </ResponseField>
    <ResponseField name="header" type="string">
      The upload portal title.
    </ResponseField>
    <ResponseField name="greeting" type="string">
      The upload portal subheader.
    </ResponseField>
    <ResponseField name="redirect" type="string">
      The URL users are redirected to after completing an upload.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="sortBy" type="object">
  The default file sort configuration for the project.

  <Expandable title="sortBy fields">
    <ResponseField name="criterion" type="string">
      The attribute files are sorted by.
    </ResponseField>
    <ResponseField name="ascending" type="boolean">
      Whether files are sorted in ascending order.
    </ResponseField>
  </Expandable>
</ResponseField>
