The Samply API is a REST interface that lets you create, read, update, and delete the core primitives that power your Samply workflow: Projects, Players, and Files. Every endpoint returns JSON, uses standard HTTP response codes, and authenticates via Bearer tokens — so you can connect from any tool that can make an HTTPS request, whether that's cURL, Postman, Zapier, or your own server-side code.

<Info>
  The Samply API is currently available on [team plans](https://samply.app/plans) and upon request. The Samply team can work with you to build custom solutions and extend the API wherever you need it.
</Info>

## Base URL

All API requests are made to the following base URL:

```
https://samply.app/api/v0
```

## Quick example

The snippet below creates a new player with comments enabled and downloads disabled — a great starting point for understanding the request shape.

```javascript
const baseUrl = "https://samply.app/api/v0";
const token = "MY-UNIQUE-TOKEN";

const response = await fetch(`${baseUrl}/players`, {
  method: "POST",
  body: JSON.stringify({
    name: "Hello World Player",
    options: {
      downloads: false,
      comments: true,
      stacks: true,
    }
  }),
  headers: {
    Authorization: `Bearer ${token}`,
    "Content-Type": "application/json"
  }
});

const player = await response.json();
```
