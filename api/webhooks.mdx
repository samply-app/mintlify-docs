Webhooks let you react to activity in your Samply account in real time. When a subscribed event occurs — such as a file finishing its upload or a new comment being posted — Samply sends an HTTP POST request to the URL you specify with a JSON payload describing the event. This makes it straightforward to trigger automations, sync external systems, or notify your team without polling the API.

## Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/webhooks` | Create a webhook |
| `DELETE` | `/webhooks/:webhookid` | Delete a webhook |

---

## Create a webhook

`POST /webhooks`

Creates a new webhook and returns a [webhook object](#webhook-object). Samply will immediately begin sending event payloads to your URL for the event types you subscribe to.

### Body parameters

<ParamField body="label" type="string" required>
  A name to help you identify this webhook in your Samply preferences.
</ParamField>

<ParamField body="url" type="string" required>
  The endpoint URL where Samply will deliver event payloads via HTTP POST.
</ParamField>

<ParamField body="events" type="string[]" required>
  An array of one or more event type strings to subscribe to. See the [available events](#available-events) table below.
</ParamField>

### Example

```javascript
const response = await fetch("https://samply.app/api/v0/webhooks", {
  method: "POST",
  headers: {
    Authorization: "Bearer YOUR_ACCESS_TOKEN",
    "Content-Type": "application/json"
  },
  body: JSON.stringify({
    label: "Upload notifier",
    url: "https://my-service.com/samply-webhook",
    events: ["upload.completed", "comment.created"]
  })
});
const webhook = await response.json();
```

---

## Delete a webhook

`DELETE /webhooks/:webhookid`

Deletes the specified webhook. Samply will stop delivering payloads to its URL immediately.

---

## Available events

Subscribe to any combination of the following event types when creating a webhook.

| Event | Description |
|-------|-------------|
| `player.created` | A new player is created |
| `player.updated` | An existing player is modified |
| `project.created` | A new project is created |
| `project.updated` | An existing project is modified |
| `comment.created` | A new comment is posted |
| `upload.started` | A file begins uploading |
| `upload.completed` | A file has finished uploading |
| `upload.failed` | A file upload fails due to an error |

---

## Webhook object

All webhook endpoints return a webhook object with the following structure.

<ResponseField name="id" type="string">
  The unique webhook id.
</ResponseField>

<ResponseField name="label" type="string">
  The human-readable name you gave this webhook.
</ResponseField>

<ResponseField name="events" type="string[]">
  The array of event types this webhook is subscribed to.
</ResponseField>

<ResponseField name="url" type="string">
  The destination URL that receives event payloads.
</ResponseField>

<ResponseField name="timeCreated" type="number">
  The time the webhook was created, in milliseconds since the Unix epoch.
</ResponseField>

<ResponseField name="timeModified" type="number">
  The time the webhook was last modified, in milliseconds since the Unix epoch.
</ResponseField>

## Tips

<Tip>
  Test your endpoint with a tool like [Webhook.site](https://webhook.site) before pointing Samply at your production URL — you can inspect incoming payloads in real time without writing any server code.
</Tip>

<Tip>
  Combine webhooks with [Zapier](https://zapier.com/apps/samply/integrations) or [Make](https://www.make.com) to trigger no-code automations — for example, posting a Slack message whenever an upload completes or a comment is added.
</Tip>

<Tip>
  You can delete or update webhooks at any time from your [Samply preferences](https://samply.app/preferences/api). There's no limit on the number of webhooks you can create.
</Tip>
